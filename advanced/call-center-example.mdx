## Overview

This example implements a complete call center solution for a health center (Centro de Salud), demonstrating how to integrate all Asterisk features into a real-world application.

## Scenario

A health center needs a call center system with:
- Public phone number: 757112233
- Business hours: Monday-Friday, 8:00-20:00
- Separate queues for registered and non-registered users
- Multiple departments: Reception, Management, Doctors, Nursing
- Database integration for user lookup
- Secure communications with TLS

## System Architecture

### Teams Structure

| Team | Extensions | Members | Pickup Group |
|------|------------|---------|-------------|
| Reception | 100-110 | 4 agents | Group 1 |
| Management | 200-209 | 3 staff | Groups 1-4 |
| Doctors | 300-309 | 3 doctors | Group 3 |
| Nursing | 400-409 | 3 nurses | Group 4 |

### Call Queues

- **Registered Users Queue**: 3 reception agents (101, 102, 103)
- **Non-registered Users Queue**: 1 dedicated agent (110)
- **Strategy**: `leastrecent` - calls go to the agent who hasn't taken a call in the longest time

## Configuration Files

### Database Schema (script.sql)

Create two tables for managing personnel and registered users:

```sql
CREATE TABLE personal(
    extension INTEGER NOT NULL,
    nombre VARCHAR(20) NOT NULL,
    equipo VARCHAR(10) NOT NULL,
    id INTEGER NOT NULL,
    PRIMARY KEY (id)
);

CREATE TABLE usuarios(
    nombre VARCHAR(20) NOT NULL,
    id INTEGER NOT NULL,
    PRIMARY KEY (id)
);

-- Sample data for staff
INSERT INTO personal VALUES(101, "MARIA", "RECEPCION", 1001);

-- Sample registered users

GRANT SELECT, INSERT, UPDATE, DELETE ON asteriskdb.* TO 'asterisk'@'localhost';
```

### Queue Configuration (queues.conf)

```ini
[general]

monitor-type = MixMonitor

[registrados]

timeout = 30
member => SIP/101
member => SIP/102
member => SIP/103

[noregistrados]
strategy = leastrecent
timeout = 30
member => SIP/110
```

### Database Functions (func_odbc.conf)

```ini

[extension]

```

## Call Flow Design

### Main IVR Menu (extensions.conf)

```ini
[exterior]
;Main entrance with business hours check
exten => 757112233,1,GotoIfTime(8:00-20:00,mon-fri,*,*?abierto,1,1)
same => n,Festival(En estos momentos no hay nadie que te pueda atender)
same => n,Festival(Nuestro horario es de 8 a 20 horas de lunes a viernes)
same => n,Hangup()

[abierto]
;IVR menu when center is open
exten => 1,1,Answer()
same => n,Festival(Bienvenido al centro de salud)
same => n(loop),Festival(Presione)
same => n,SayNumber(1)
same => n,Festival(si es un usuario registrado o)
same => n,SayNumber(2)
same => n,Festival(en caso contrario)
same => n,Read(opcion,,1,3)
same => n,SayNumber(${opcion})
same => n,GotoIf($["${opcion}" = "1"]?10,1)
same => n,GotoIf($["${opcion}" = "2"]?20,1)
same => n,Festival(Opcion incorrecta)
same => n,Goto(1,loop)
same => n,Hangup()

;Registered user path
exten => 10,1(id),Festival(Por favor marque su numero de identificacion o presione)
same => n,SayNumber(9)
same => n,Festival(para volver al menu)
same => n,Read(id,,4,3,5)
same => n,SayNumber(${id})
same => n,GotoIf($["${id}" = "9"]?1,loop)
same => n,Set(nombre=${ODBC_usuario(${id})})
same => n,GotoIf($["${nombre}" != ""]?recepcion,1,1)
same => n,Festival(Usuario no encontrado)
same => n,Goto(10,id)
same => n,Hangup()

;Non-registered user path
exten => 20,1,Goto(recepcion,2,1)
same => n,Hangup()
```

### Reception Queue Context

```ini
[recepcion]
;Queue for registered users
exten => 1,1,Set(CHANNEL(musicclass)=recepcion)
same => n,Answer()
same => n,Festival(Hola ${nombre})
same => n(reg),Festival(Por favor espere a que le atiendan)
same => n,Queue(registrados,,,,60)
same => n,Goto(1,reg)
same => n,Hangup()

;Queue for non-registered users
exten => 2,1,Set(CHANNEL(musicclass)=recepcion)
same => n,Answer()
same => n(noreg),Festival(Por favor espere a que le atiendan)
same => n,Queue(noregistrados,,,,60)
same => n,Goto(2,noreg)
same => n,Hangup()

;Call another extension
exten => _XXX,1,Set(extension=${EXTEN})
same => n,Set(nombre=${ODBC_extension(${extension})})
same => n,GotoIf($["${nombre}" != ""]?llamar,${extension},1)
same => n,Festival(La extension ${extension} no se encuentra disponible)
same => n,Hangup()

;Voicemail access
exten => _*XXX,1,Set(llamante=${CALLERID(num)})
same => n,Goto(buzon-voz,1,1)
same => n,Hangup()

;Call pickup
exten => _*8.,1,Set(extension=${EXTEN:2})
same => n,Set(pref=${extension:0:1})
same => n,GotoIf($["${pref}" = "1"]?llamar,8,1)
same => n,Festival(No tiene permiso para capturar una llamada a la extension ${extension})
same => n,Hangup()
```

### Group Calling

### Voicemail with PIN Verification

```ini
[buzon-voz]
exten => 1,1,Festival(Para acceder a su buzon de voz)
same => n(pin),Festival(Marque su numero de identificacion privado)
same => n,Read(pin,,4,3,5)
same => n,Set(idpin=${ODBC_pin(${llamante})})
same => n,GotoIf($["${pin}" = "${idpin}"]?2,1)
same => n,Festival(Numero de identificacion erroneo)
same => n,Goto(1,pin)
same => n,Hangup()

exten => 2,1,Festival(Marque 1 para continuar)
same => n,VoiceMailMain(${llamante})
same => n,Hangup()
```

## Key Features Demonstrated

### 1. Time-Based Routing

Calls outside business hours receive an automated message:

```ini
exten => 757112233,1,GotoIfTime(8:00-20:00,mon-fri,*,*?abierto,1,1)
```

### 2. Database Integration

User lookup and verification using MariaDB:

```ini
Set(nombre=${ODBC_usuario(${id})})
```

### 3. Call Queuing

Automatic call distribution with hold music:

```ini
Queue(registrados,,,,60)
```

### 4. Call Pickup Groups

Team members can pickup calls within their group:

```ini
callgroup=1
pickupgroup=1
```

Management can pickup from any group:

```ini
callgroup=2
pickupgroup=1-4
```

### 5. Text-to-Speech (Festival)

Dynamic voice messages:

```ini
Festival(Hola ${nombre})
```

### 6. Secure Communications

All extensions use TLS:

```ini
transport=tls
```

### 7. Voicemail with Security

PIN verification before accessing voicemail:

```ini
Set(idpin=${ODBC_pin(${llamante})})
GotoIf($["${pin}" = "${idpin}"]?2,1)
```

## Testing the Call Center

### Test Scenario 1: Registered User Call

1. Call 757112233 during business hours
2. Press 1 for registered user
3. Enter valid ID (e.g., 50 for JULIA)
4. Wait in queue to be connected to reception agent

### Test Scenario 2: Non-Registered User

1. Call 757112233
2. Press 2 for non-registered user
3. Connect to dedicated agent 110

### Test Scenario 3: Internal Extension

From extension 101:
1. Dial 301 to call a doctor
2. Or dial 300 to ring all doctors simultaneously

### Test Scenario 4: Call Pickup

When extension 102 is ringing:
1. From extension 101, dial *8102
2. The call is transferred to 101

## Production Considerations

### Monitoring and Logging

- Enable CDR (Call Detail Records) for call tracking
- Configure queue logging for performance metrics
- Set up alerts for queue overflow

### Scalability

- Add more agents to queues as needed
- Consider dynamic queue membership
- Implement callback functionality for long wait times

### Redundancy

- Set up failover to backup PBX
- Database replication for high availability
- Multiple reception agents to handle peak loads

### Security

- Regular password rotation
- TLS certificate renewal
- Monitor for unusual call patterns
- Implement fail2ban for brute force protection
